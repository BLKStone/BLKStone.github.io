尝试修复引导

安装过程中多次遇到grub-install /dev/sda failed，尝试修复引导并没有解决，最终更换为64位 Ubuntu Kylin版的镜像后成功（之前用的是32位的xubuntu）。

Ubuntu 使用时的问题

deja-dup-monitor 占用大量内存

ubuntu开机时，deja-dup-monitor占用大量内存，导致读写swap分区。

解决办法 进入ubuntu 软件中心，搜索deja，直接卸载该软件。

可以参考这里的回答,原因似乎是chrome-remote-desktop与该软件冲突产生的一个bug。
